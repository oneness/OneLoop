Evidence Agent pattern

In orchestration modes, providers never get direct tool access. They get one tool: request_evidence. They ask, the main agent gathers.

Evidence is deduplicated (same query = cached result) and shared across all providers in the same orchestration. Shell commands pass a safety check (read-only inspection only: find, grep, git log, etc.).

Sessions & Resilience

📂 Append-only sessions

Every message, tool call, and result is appended to a JSONL file. No mutation, no corruption.

.oneloop/sessions/2025-07-11.jsonl
.oneloop/sessions/2025-07-11-001.jsonl

🔄 Auto-compaction

When context nears the model's limit, the agent compacts: generates a structured summary, rotates to a new session, and replays recent user messages verbatim.

Triggered at 85% of context window. Preserves continuity.

Resilience features

FeatureBehavior
Retry with fallback On provider failure: retries 3×, then interactively offers alternative providers
Auto-compaction Summarizes and rotates session at 85% context — no lost context
Parallel tools Multiple tool calls in one response execute concurrently
Output truncation Large tool outputs are truncated before re-entering context
Stop anywhere Ctrl+C stops the current request cleanly; session is preserved
AGENTS.md Loaded as system prompt if present in the project directory
