Directive System

One pattern: #!directive words#! followed by your message.
Parsed before any model call. The model never sees #!.

Syntax

#!directive words#! your message here

Directive words are space-separated tokens between the #! markers. They control which providers run and how they're orchestrated. Everything after the closing #! is the user message sent to the model(s).

Examples

Directive Patterns

#!anthropic#! review this function
Single provider. Routes to Anthropic with full agent loop — tools, session history, auto-compaction.
#!anthropic openai#! should we use Lua?
Multi-provider → consensus. Two+ providers without an explicit mode defaults to consensus. Both answer in parallel, then a judge synthesizes.
#!consensus anthropic openai zai judge:openai#! Should we ship plugins before skills?
Consensus with judge. All three answer independently. OpenAI writes the final synthesis.
#!debate anthropic openai zai rounds:2 judge:anthropic#! Should we add hooks?
Debate with 2 rounds. Initial answers → 2 rounds of critique/revision → judge synthesis.
#!anthropic openai tools:none#! compare these approaches
Explicit no tools. By default consensus/debate providers get read and web_search so they can gather evidence. Use tools:none to disable.
#!anthropic format:md#! summarize this file
Output format. Requests markdown-formatted output. Also supports format:html.
#!format:md#! summarize the project
No provider specified. Uses the default provider selection order (Z.AI → OpenAI → Anthropic) with markdown output format.
